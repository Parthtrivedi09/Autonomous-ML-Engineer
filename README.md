1. Introduction
Data has become one of the most valuable assets in the modern world, but raw data alone has little value until it is transformed into meaningful insights. In practice, nearly 70–80% of a data scientist's effort is spent not on building machine learning models, but on understanding, cleaning, preprocessing, engineering features, selecting suitable models, tuning hyperparameters, and evaluating results.
Although several AutoML platforms exist today, most of them operate as black boxes. Users upload a dataset and receive a trained model with little or no explanation regarding the decisions made during the process. This lack of transparency makes it difficult to trust, customize, or learn from the workflow.
AutoML Studio addresses this problem by introducing an LLM-Orchestrated Automated Machine Learning Platform that combines the reasoning capabilities of Large Language Models with the deterministic execution of Python-based machine learning libraries. Instead of replacing the data scientist, AutoML Studio acts as an intelligent AI assistant that explains every recommendation, seeks human approval, and automates the complete end-to-end machine learning workflow.
________________________________________
2. Project Overview
AutoML Studio is an Agent-Based AutoML platform where each phase of the machine learning lifecycle is handled by specialized AI agents.
Unlike traditional conversational AI systems, the LLM is never responsible for manipulating the dataset directly.
Instead, the project follows a strict separation of responsibilities:
•	Python performs all computation and execution
•	The LLM performs reasoning and decision making
•	The user validates every important decision
This architecture ensures that every transformation performed on the dataset is deterministic, reproducible, and explainable.

4. Why Not Simply Upload the Dataset to ChatGPT?
A common question is:
Why build AutoML Studio when ChatGPT can already analyze CSV files?
The answer lies in scalability, determinism, and workflow orchestration.
4.1 Context Window Limitation
Large Language Models have finite context windows.
Consider a dataset containing:
•	100,000 rows
•	50 columns
Such a dataset contains millions of individual values, translating to several million tokens—far exceeding the context limits of current LLMs.
Problems include:
•	Cannot fit into the model's context window
•	Extremely expensive inference
•	Slow processing
•	Loss of important statistical patterns
•	Increased hallucination risk
Instead, AutoML Studio allows Python to process the dataset and compute essential statistics. Only these compact summaries are passed to the LLM for reasoning.
________________________________________
4.2 LLMs Are Better at Reasoning Than Computation
Large Language Models excel at:
•	understanding reports
•	identifying patterns
•	suggesting preprocessing strategies
•	recommending algorithms
•	explaining decisions
They are not designed to:
•	iterate over millions of rows
•	compute statistics efficiently
•	execute preprocessing pipelines
•	train machine learning models
Therefore,
Python performs computation.
The LLM performs reasoning.
________________________________________
4.3 ChatGPT Is Not a Workflow Engine
Uploading a dataset into ChatGPT produces only a conversation.
AutoML Studio provides:
•	modular AI agents
•	workflow state management
•	deterministic execution
•	structured JSON communication
•	user validation
•	experiment reproducibility
•	extensibility
•	retrieval-augmented knowledge (RAG)
•	complete ML lifecycle automation

Example:
[
    {
        "operation":"fill_missing",
        "column":"Age",

9. Technical Challenges Faced
Developing AutoML Studio presented several real-world engineering challenges.
Duplicate Detection Issue
While testing the data cleaning pipeline, a critical issue emerged.
The cleaning agent removed approximately 116 rows from one of the testing datasets.
Initially, these rows appeared to be duplicates.
However, investigation revealed that columns such as Name and ID had been removed before duplicate detection.
Once these identifying columns were deleted, several distinct records became identical across the remaining features.
Python therefore classified them as duplicates and removed them, resulting in unintended information loss.
This highlighted an important lesson:
Two rows may appear identical after preprocessing while still representing different real-world entities.
The duplicate detection strategy was redesigned to preserve the semantic integrity of the dataset.
________________________________________
Rule-Based Complexity
Another challenge is handling the wide variety of data quality issues encountered across different datasets.
Each new dataset introduces unique problems, such as:
•	inconsistent categorical values
•	mixed data types
•	incorrect missing-value representations
•	high-cardinality categorical features
•	skewed distributions
•	outliers
•	multicollinearity
•	target leakage
•	identifier columns
•	date parsing
•	rare categories
Currently, many of these cases require dedicated Python functions.
As the platform evolves, the objective is to replace handcrafted rules with more adaptive and intelligent preprocessing strategies.

Model training and hyperparameter optimization are currently performed on a local machine.
Large datasets significantly increase computation time.
Future improvements include:

13. What Makes AutoML Studio Different?
AutoML Studio is not merely an interface around an LLM, nor is it a conventional AutoML system. Its distinguishing feature lies in the clear separation of reasoning and execution. Rather than allowing an LLM to manipulate raw datasets or blindly automate decisions, the platform relies on Python for deterministic analysis and execution, while the LLM interprets analytical reports, generates context-aware recommendations, and explains its reasoning in natural language. Every recommendation is reviewed through a human-in-the-loop validation step before any action is taken, ensuring transparency, trust, and user control. The use of specialized agents, structured JSON communication, and a modular workflow further makes the system reproducible, extensible, and significantly more explainable than traditional AutoML frameworks or one-off interactions with conversational AI.
